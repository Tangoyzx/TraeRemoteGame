# 技能系统框架

本文档定义类吸血鬼生存者项目的技能/升级系统框架。它是所有技能平衡的**方法论基准**，相对稳定；具体技能的数值校准见 [skill-calibration.md](./skill-calibration.md)。

## 1. 设计目标

- **同一通用升级，在不同技能上有不同具体效果，但价值大致相当。**
- 升级点改的是"通用属性"，技能只关心通用属性，不关心升级叫什么名字。
- 平衡只需调"通用属性 → 技能参数"这一层映射，而非每个升级 × 每个技能单独调。

## 2. 核心架构:通用属性层 + 技能适配器

```
[升级点 Count+1]  ─┐
                   ├──> 通用属性层 (StatBlock)  ──> 技能适配器 apply_stat()  ──> 技能具体参数
[升级点 Frequency+15%] ─┘
```

- **通用属性层**:一个跨技能共享的属性集合(见 §3)。升级点修改的是这些属性的"堆叠次数"。
- **技能适配器**:每个技能实现 `apply_stat(stat, stacks)`,把通用属性翻译成自己的内部参数(冷却、转速、伤害、数量等),并乘以该技能的**适配系数** `k`。
- **边际递减**:全局统一的曲线,防止堆单一属性爆表(见 §5)。

### 适配器签名(GDScript 草图)

```gdscript
# 通用属性枚举
enum Stat { COUNT, FREQUENCY, DAMAGE, AREA, PIERCE, DURATION, SPEED }

# 每个技能基类
class_name Weapon
extends Node

# 子类实现:把通用属性堆叠次数翻译成自己的参数
func apply_stat(stat: int, stacks: int) -> void:
    var magnitude := StatMath.effective_magnitude(stat, stacks)  # 全局边际递减
    var k := _stat_coefficient(stat)                              # 本技能的适配系数
    match stat:
        Stat.COUNT:       _apply_count(magnitude * k)
        Stat.FREQUENCY:   _apply_frequency(magnitude * k)
        Stat.DAMAGE:      _apply_damage(magnitude * k)
        # ...
    _refresh_params()

# 子类提供系数表
func _stat_coefficient(stat: int) -> float:
    return 1.0   # 默认,子类覆盖
```

## 3. 通用属性定义

| 属性 | 含义 | 单位 | 说明 |
|---|---|---|---|
| `COUNT` 数量 | 实体数量 | +1 实体 | 子弹多发 / 剑多把 / 分裂体多个 |
| `FREQUENCY` 频率 | 触发速率 | +% | 冷却 ↓ 或转速 ↑ |
| `DAMAGE` 伤害 | 单次打击量 | +% | 单次命中伤害 |
| `AREA` 范围 | 覆盖/尺寸 | +% | 体积、半径、溅射、旋转半径 |
| `PIERCE` 穿透 | 命中能力 | +1 / -冷却 | 子弹穿透敌人数 / 剑命中冷却 ↓ |
| `DURATION` 持续 | 存在时间 | +% | 子弹寿命、召唤体存活时间 |
| `SPEED` 速度 | 移动速度 | +% | 弹速;对无此概念的技能折叠进 FREQUENCY |

> **折叠规则**:某通用属性对某技能无独立意义时,可折叠进另一属性(如 OrbitSword 的 SPEED → FREQUENCY),而非留空。详见 §7。

## 4. 平衡方法论

### 4.1 第一锚点:DPS

每次升级对每个技能的 **DPS 增益**应落在同一目标区间(建议单次升级 **+10%~+25% DPS**)。先算 DPS,再算覆盖和手感。

### 4.2 三维增益

DPS 相同不代表价值相同,需同时盯三个维度:

- **DPS 增益**:输出总量
- **覆盖增益**:能打到的目标数 / 区域
- **手感增益**:视觉密度、压制力、操作感

同一升级对两个技能若 DPS 增益相同,但其中一方还附带覆盖或手感增益,则需在适配系数 `k` 上对**附带额外收益的一方**打折(见 [skill-calibration.md](./skill-calibration.md) §6)。

### 4.3 典型陷阱:Count 的不对称

`COUNT +1` 是最容易失衡的属性,因为它对"子弹型"和"环绕型"技能的增益类型完全不同:

| 技能类型 | COUNT +1 的增益类型 | 单体 DPS | 群体 DPS |
|---|---|---|---|
| 子弹型(单发,瞄准单目标) | 覆盖增益(多瞄一个目标) | 不变 | ×2 |
| 环绕型(实体在场上持续) | DPS 增益(多一个实体打同一目标) | ×2 | ×2 |

→ 表面都是"+1 实体",但环绕型白赚了单体 DPS。**校准策略**:Count 保持"字面 +1 实体"(直觉一致),但通过 (a) 全局边际递减更陡、(b) 升级池稀有度更低、(c) 适配系数微调 三者联合压制。详见 [skill-calibration.md](./skill-calibration.md) §7。

### 4.4 典型安全项:Frequency

`FREQUENCY +x%` 对绝大多数技能都是**纯线性 DPS 增益**(冷却 ↓ → DPS ↑;转速 ↑ → DPS ↑),天然平衡。可作为新技能上线的"基准属性"。

## 5. 边际递减(全局,按属性)

同一通用属性在同一技能上重复堆叠时,收益递减。曲线**全局统一**,按属性区分陡峭度:

```
effective_magnitude(stat, stacks) = base_per_stack(stat) / (1 + dr_divisor(stat) * (stacks - 1))
```

| 属性 | `dr_divisor` | 堆叠 1/2/3/4/5 的收益比 |
|---|---|---|
| COUNT | 0.5 | 100% / 67% / 50% / 40% / 33% |
| FREQUENCY | 0.4 | 100% / 71% / 56% / 45% / 38% |
| DAMAGE | 0.4 | 100% / 71% / 56% / 45% / 38% |
| AREA | 0.4 | 100% / 71% / 56% / 45% / 38% |
| PIERCE | 0.5 | 100% / 67% / 50% / 40% / 33% |
| DURATION | 0.4 | 100% / 71% / 56% / 45% / 38% |
| SPEED | 0.4 | 100% / 71% / 56% / 45% / 38% |

- COUNT / PIERCE 用更陡的 0.5,因为它们是"整数 +1"型,堆叠爆炸风险高。
- 单属性单技能堆叠上限 **5 层**,防止极端 build。

## 6. 适配系数表(模板)

每个技能 × 每个通用属性一个系数 `k`,默认 1.0。`k < 1` 表示该技能从该属性获得额外隐性收益,需打折;`k > 1` 表示该属性对该技能偏弱,可加成。**具体数值见 [skill-calibration.md](./skill-calibration.md) §6。**

校准 `k` 的流程:
1. 全部 `k = 1.0` 起步。
2. 算每个升级对每个技能的 DPS 增益。
3. 若某组合显著超出 +10%~+25% 区间,且原因是附带覆盖/手感增益,则下调 `k`。
4. 跑极端 build 测试(单属性堆满 5 层 + 跨技能协同)。

## 7. 无效翻译处理

某通用属性对某技能无意义时(如 OrbitSword 无 SPEED、AutoShooter 的 DURATION 边际很低),三种处理,按优先级:

1. **折叠**:翻译成相近属性(OrbitSword SPEED → FREQUENCY)。首选。
2. **替代效果**:给一个等价但不同名的效果(剑的 PIERCE → 命中冷却 ↓)。
3. **禁用**:UI 标灰,不投放。最差体验,尽量少用。

> 原则:**让每个通用升级对每个技能都有可感知的效果**,这是"同一描述不同反馈"设计成立的前提。

## 8. 升级投放与稀有度

升级点在升级界面以**加权随机池**投放。权重控制平衡,而非靠效果数值:

- **基础权重** 100:FREQUENCY、DAMAGE(线性、安全)
- **中等权重** 60:AREA、DURATION、SPEED
- **低权重** 35:COUNT、PIERCE(整数 +1 型,爆炸风险高)

升级未解锁对应武器时,该武器的所有 stat 升级不入池。

## 9. 术语表

| 术语 | 定义 |
|---|---|
| 通用属性 (Stat) | 跨技能共享的抽象属性,如 COUNT、FREQUENCY |
| 适配器 (Adapter) | 技能内把通用属性翻译成具体参数的函数 `apply_stat` |
| 适配系数 `k` | 每技能 × 每属性的平衡系数 |
| 边际递减 (DR) | 同属性重复堆叠收益递减 |
| DPS 锚点 | 以单次升级的 DPS 增益为平衡第一指标 |
| 三维增益 | DPS / 覆盖 / 手感 三个平衡维度 |
| 折叠 | 把无意义的通用属性翻译进相近属性 |
